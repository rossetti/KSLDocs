KSLCore/ksl.modeling.supplychain.cost/CostParams

CostParams

data class CostParams(val carryingRate: Double = 0.1, val backorderRate: Double = 0.0, val orderingCost: Double = 5.5, val unloadingCost: Double = 30.0, val loadingCost: Double = 40.0, val shippingCost: Double = 15.0, val stockoutCost: Double = 0.0, val lostSaleCost: Double = 0.0, val unitShortageCost: Double = 0.0, val esLoadingCost: Double = 40.0)(source)

Parameter bundle for a DefaultMultiEchelonCostFormulation. All rate values are denominated in 1 / (modeler-chosen time unit) — the framework performs no time-unit conversion and makes no presumption about what the modeler's rate basis is. The emitted cost-line Responses are therefore reported in the modeler's chosen time unit (a carryingRate of 0.10/year produces a holding cost in $/year; 0.10/day produces it in $/day).

Discrete-event $/event values are dimensionless with respect to time and emit a per-replication total.

Parameters

carryingRate

continuous-rate inventory carrying rate per unit per (modeler-chosen time unit). Multiplies the time-weighted on-hand and on-order averages.

backorderRate

continuous-rate backlog carrying rate per unit per (modeler-chosen time unit). Multiplies the time-weighted backlog average. Default 0.0 disables the continuous backorder-cost line.

orderingCost

$ per replenishment-order event.

unloadingCost

$ per inbound shipment.

loadingCost

$ per outbound shipment.

shippingCost

$ per outbound shipment.

stockoutCost

$ per stockout event.

lostSaleCost

$ per lost-sale event.

unitShortageCost

$ per unit short.

esLoadingCost

$ per outbound shipment from the external supplier.
